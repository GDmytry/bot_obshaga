# Smart-Общага 502 🏠

Единая экосистема управления локальной сетью комнаты, финансового учета совместных расходов и автоматизированного контроля дисциплины.

## 🛠 Технический стек и архитектура

Система построена по модульному принципу:
- **Telegram Бот**: `aiogram 3.x` (Асинхронный фреймворк)
- **Веб-дашборд**: `FastAPI` + `Uvicorn` (Отвечает за визуализацию данных, порт `8000`)
- **База данных**: Поддержка двух режимов:
  - `PostgreSQL` (через `asyncpg`) — основной (Production) режим.
  - `SQLite` (через `aiosqlite`) — режим разработки (активируется переменной `DEV_MODE=1`).
- **Фоновые задачи**: `APScheduler` (Отвечает за проверку присутствия по сети, снятие наказаний и еженедельные отчеты).

### Основные модули системы:
1. **bot/handlers/** — Обработчики команд Telegram.
   - `common.py` — /start, /register, /status, /addmac.
   - `finance.py` — Умный Общак (/buy, /balance).
   - `queues.py` — Очереди (/q).
   - `discipline.py` — Выговоры (/warn, /unwarn).
2. **database/** — Работа с БД.
   - `connection.py` — Универсальный коннектор (PostgreSQL/SQLite).
   - `init_db.py` — Инициализация таблиц.
3. **modules/** — Логика системы.
   - `presence/worker.py` — Пингует роутер Huawei AX2 (по SSH или через API), проверяет, кто дома по MAC-адресам устройств.
   - `finance/scheduler.py` — Отправляет еженедельные финансовые отчеты по понедельникам.

## 🚀 Как перенести бота на сервер (Debian Linux)

На сервере (старом ноутбуке) мы будем использовать **Docker** и **Docker Compose**. Это самый надежный способ, так как он сам поднимет и базу данных PostgreSQL, и самого бота.

### Шаг 1. Подготовка сервера
1. Установите чистый Debian на ноутбук.
2. Установите Docker:
   ```bash
   sudo apt-get update
   sudo apt-get install -y docker.io docker-compose
   ```
3. Разрешите запуск Docker без sudo (опционально):
   ```bash
   sudo usermod -aG docker $USER
   newgrp docker
   ```

### Шаг 2. Перенос файлов проекта
Перенесите папку с ботом на сервер. Это можно сделать через GitHub, флешку или по SSH (через `scp`). 
Убедитесь, что перенесли файлы: `main.py`, `config.py`, `Dockerfile`, `docker-compose.yml`, `requirements.txt` и все папки (`bot/`, `database/`, `modules/`).
*(Папку `venv` переносить НЕ НУЖНО!)*

### Шаг 3. Настройка окружения на сервере
В папке с проектом на сервере создайте файл `.env`:
```bash
nano .env
```
Добавьте туда настройки:
```ini
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_IDS=ваш_telegram_id
DEV_MODE=0
# Ngrok использовать не обязательно, можно настроить прямо на сервере или через белые IP.
```
*Обратите внимание: `DEV_MODE=0`, так как мы будем использовать настоящую базу PostgreSQL в Docker.*

### Шаг 4. Запуск!
В папке с проектом выполните команду:
```bash
docker-compose up -d --build
```
Эта команда:
1. Скачает PostgreSQL 15.
2. Соберет контейнер с ботом (по `Dockerfile`).
3. Запустит их оба. Бот сам создаст все нужные таблицы при первом запуске.

### 💡 Полезные команды на сервере:
- **Посмотреть логи бота:**
  ```bash
  docker-compose logs -f bot
  ```
- **Остановить бота:**
  ```bash
  docker-compose down
  ```
- **Перезапустить после изменений кода:**
  ```bash
  docker-compose up -d --build
  ```

## 🌐 Дашборд
Когда бот запущен на сервере, дашборд будет доступен по локальному адресу ноутбука: `http://<IP_НОУТБУКА>:8000/`. Вы можете пробросить этот порт наружу через роутер или использовать Ngrok прямо на сервере (запустив его аналогично `start.bat`).
